# 🎮 任务 01：学习 Git 和 GitHub

> 这是一个**闯关游戏**！一共 9 关，一关一关来，你就学会怎么用 Git 和 GitHub「交作业」啦。

## 🎯 这次你会学会

- 把老师的项目 **fork**（复制一份到自己账号）🍴
- 把项目 **clone**（下载到自己电脑）⬇️
- 用 **Git** 给作品「存档」📀
- 把作品 **push**（上传）到网上 ☁️
- 建立一个 **PR**，把作业「交」给老师看 📬

## 🤔 先搞懂几个词

**Git** 是什么？

> 你玩游戏时会「**存档**」对吧？这样万一失败了，还能读回上一次的进度。
> Git 就是给你的文件做「存档」的工具。每存一次档，就叫一次「**commit（提交）**」。

**GitHub** 是什么？

> Git 的存档一开始只在你自己的电脑上。
> GitHub 是一个网站，可以把你的存档**传到网上**，这样别人也能看到，还能一起做项目。就像把照片传到网上的相册一样 📸。

**Fork** 和 **Clone** 又是什么？（这次的重点！）

> 老师有一份项目放在 GitHub 上。但你**不能直接改老师的东西**，得先自己弄一份。
>
> - **Fork（复印）** 🍴：在 GitHub 网页上，把老师的项目**整个复制一份到你自己的账号**，就像把老师的练习册**复印一本**，变成「你自己的」那一本。
> - **Clone（下载）** ⬇️：把你自己账号里的那一份**下载到电脑上**，这样你才能在电脑上写字、做作业。

**这几个地方是什么关系？**

```
老师的 GitHub 仓库
        │
        │ 1. Fork：复制到你的 GitHub 账号
        ▼
你的 GitHub 仓库
        │
        │ 2. Clone：下载到你的电脑
        ▼
你的电脑文件夹
        │
        │ 3. Commit：在电脑里存档
        │ 4. Push：上传到你的 GitHub 仓库
        ▼
你的 GitHub 仓库
        │
        │ 5. PR：请老师把你的作业收回老师仓库
        ▼
老师的 GitHub 仓库
```

记住一句话：**先复制老师的，再下载自己的；写完先存档，再上传；最后用 PR 交给老师。**

## 🧰 开始前，先准备好

1. 电脑上已经装好了 **Git**。在终端（黑黑的命令窗口）里输入下面这行，能看到版本号就说明装好了：

   ```bash
   git --version
   ```

2. 有一个 **GitHub 账号**（网址：https://github.com ）。没有的话，请大人帮忙注册一个。

3. 知道老师的项目网址（这个项目的网址）。

> 💡 小提示：命令里的字母、空格、符号都要一模一样哦，就像抄口令一样。

---

## 🏁 关卡 1：Fork —— 复印一份到自己账号 🍴

1. 用浏览器打开**老师的项目网页**。
2. 点右上角的 **「Fork」** 按钮。
3. 稍等几秒，GitHub 就会在**你自己的账号**下面，生成一份一模一样的项目。

做完后，看看浏览器地址栏，网址会变成：

```
https://github.com/你的用户名/pbl-courses
```

看到是**你自己的用户名**，就成功啦！🎉 这一份就是「你的复印本」。

---

## 🏁 关卡 2：Clone —— 下载到自己电脑 ⬇️

1. 在**你自己**那份项目的网页上，点绿色的 **「Code」** 按钮。
2. 选 **HTTPS**，复制那个网址（长得像 `https://github.com/你的用户名/pbl-courses.git`）。
3. 打开终端，输入（把网址换成你刚才复制的）：

   ```bash
   git clone https://github.com/你的用户名/pbl-courses.git
   ```

4. 下载好后，进入这个项目文件夹：

   ```bash
   cd pbl-courses
   ```

🎉 现在，整个项目就在你的电脑里啦！

---

## 🏁 关卡 3：开一条自己的「小路」（新建分支）

做作业前，先开一条自己的「**小路**」，在小路上做，不影响主路。这条小路就叫「**branch（分支）**」。

新建并切换到一条带有你 **GitHub 用户名** 的小路，方便老师分辨是谁的作业。比如你的 GitHub 用户名是 `xiaoming`，分支就叫 `my-card-xiaoming`。

规则：

- 分支名必须包含你的 GitHub 用户名。
- 推荐格式：`my-card-你的GitHub用户名`。

```bash
git checkout -b my-card-xiaoming
```

> 把上面的 `xiaoming` 换成你自己的 GitHub 用户名。

用这个命令看看自己现在站在哪条路上：

```bash
git branch
```

前面有个 `*` 星号的，就是你现在所在的分支，应该是你刚才新建的 `my-card-xiaoming`（把 `xiaoming` 换成你的 GitHub 用户名）。🌿

> 🌳 想象一棵大树：`main` 是主干，`my-card-xiaoming` 是你新长出来的一根小树枝。你可以在小树枝上安心尝试，不怕弄坏主干。

## 🏁 关卡 4：做一张「自我介绍卡片」

先在 `tasks/01-learn-git/work/` 里面给自己建一个专属文件夹。

规则：

- 文件夹名必须包含你的 GitHub 用户名。
- 推荐直接使用你的 GitHub 用户名，比如 `xiaoming`。
- 不要把作品直接放在 `work/` 根目录里。

比如你的 GitHub 用户名是 `xiaoming`，可以这样建：

```bash
mkdir -p tasks/01-learn-git/work/xiaoming
```

然后在自己的文件夹里新建一个文件叫 `about-me.md`，写上你的自我介绍。比如：

```
tasks/01-learn-git/work/xiaoming/about-me.md
```

可以照着下面这个样子写（换成你自己的内容）：

```markdown
# 你好，我是小明！👋

- 🎂 我今年 9 岁
- 🎨 我最喜欢：画画、踢足球
- 🐶 我最喜欢的动物是：小狗
- 🌟 我长大想当：宇航员
```

写好保存！这就是你这次要提交的**作品**。🎉

## 🏁 关卡 5：看看 Git 现在的状态

在终端里输入这个命令，问问 Git：「现在有什么新变化呀？」

```bash
git status
```

你应该会看到 Git 说：你的专属文件夹里有一个新文件 `about-me.md` 还没有被保存（它可能显示成红色）。

> 🔍 `git status` 就像一面镜子，随时告诉你「哪些东西改了、哪些还没存档」。这是最常用的命令，随时都可以敲一敲看看情况！

---

## 🏁 关卡 6：把作品「存档」（add + commit）

存档要分两步：

**第一步：告诉 Git「我要存这些变化」**

```bash
git add .
```

> 那个 `.`（小点）的意思是「所有的改动都算上」。再敲一次 `git status`，你会发现文件变成绿色啦。

**第二步：正式存档，并写一句「这次存了什么」**

```bash
git commit -m "加入我的自我介绍卡片"
```

> ✏️ 引号里的话叫「**commit 信息**」，用来记录你这次做了什么。就像给存档起个名字，以后一看就知道。

🎉 恭喜！你完成了人生第一次 **commit**！

---

## 🏁 关卡 7：把作品 push 到网上 ☁️

把你的存档「推」到 GitHub 上**你自己**那份项目里：

```bash
git push -u origin my-card-xiaoming
```

> 把上面的 `xiaoming` 换成你自己的 GitHub 用户名。

> ⚠️ 第一次 push 时，GitHub 可能会让你登录、验证身份，请大人帮忙一下（见文件最下面的〈**大人帮忙区**〉）。

推成功后，去你自己那份项目的网页刷新一下，就能看到 `my-card-xiaoming` 这条小路和你的作品啦！☁️✨

> 这里的 `xiaoming` 也要换成你自己的 GitHub 用户名。

> 🚀 `push` 就是「推上去」的意思——把你电脑里的存档，推到网上。

---

## 🏁 关卡 8：建立 PR —— 把作业「交」给老师 📬

**PR** 的全名是 **Pull Request**。在这里，它的意思就是：

> 「老师，我在自己的复印本上做好作业啦，**请你看一看、收下吧！**」🙋

怎么建 PR：

1. 打开**你自己**那份项目的 GitHub 网页。
2. 页面上一般会出现一个提示：**「Compare & pull request」**，点它。
   - 如果没看到，就点 **「Contribute」** → **「Open pull request」**。
3. 检查一下方向：是把 **你的 `my-card-你的GitHub用户名`** 合并到 **老师项目的 `main`**（页面上会显示两个仓库的名字，箭头从你的指向老师的）。
4. 写个标题，比如：`加入我的自我介绍卡片`。
5. 点绿色按钮 **「Create pull request」**。

### ✅ PR 方向检查

点绿色按钮前，先检查这几项：

- **base repository** 应该是老师的仓库：`laozhu/pbl-courses`
- **base branch** 应该是老师的主分支：`main`
- **head repository** 应该是你的仓库：`你的GitHub用户名/pbl-courses`
- **compare branch** 应该是你的分支：`my-card-你的GitHub用户名`

如果方向反了，不要点创建按钮。先请老师或家长帮你一起检查。

🎉 太厉害了！你把作业**提交给老师**了，这正是程序员团队合作时**每天都在做的事**！

## 📔 魔法命令小抄（贴在墙上随时看）

| 命令 | 作用 | 一句话记忆 |
|------|------|-----------|
| `git clone 网址` | 把网上的项目下载到电脑 | 「搬回家」⬇️ |

## 🧯 卡住了怎么办？

先别着急。Git 和 GitHub 第一次用，卡住很正常。你可以按下面的方法检查。

| 卡住的地方 | 先检查什么 |
|------------|------------|
| 找不到 Fork 按钮 | 确认你打开的是老师的项目网页，不是自己的项目网页 |
| `git clone` 失败 | 确认网址来自你自己的仓库，并且以 `.git` 结尾 |
| 终端说找不到文件夹 | 先输入 `pwd` 看看自己在哪里，再输入 `ls` 看看有哪些文件夹 |
| `git commit` 失败 | 如果提示 name 或 email，请看下面的〈大人帮忙区〉设置 Git 名字和邮箱 |
| `git push` 要求登录 | 请大人帮忙完成 GitHub 登录或验证 |
| 找不到 PR 按钮 | 打开你自己的 GitHub 仓库，进入刚 push 的分支，再找 **Contribute** |
| 不确定 PR 方向 | 回到关卡 8 的「PR 方向检查」，确认老师仓库在 base，你的仓库在 head |

如果还是不行，把终端里最后 3 行提示复制下来，发到 issue 或微信群里请老师看。

## 👨‍👩‍👧 大人帮忙区（给家长/老师）

如果孩子是第一次用 Git，下面这些可以由大人帮忙：

1. **设置 Git 的名字和邮箱**（clone 完后，只需设置一次）：
   ```bash
   git config --global user.name "孩子的名字"
   git config --global user.email "邮箱@example.com"
   ```

2. **身份验证**：GitHub 现在需要用**浏览器登录**或 **Personal Access Token** 来验证身份。第一次 `push` 时会弹出来要求登录，请协助孩子完成。（装了 GitHub 官方的 `gh` 命令行工具会更省事。）

3. **关于 fork 工作流**：孩子 fork 的是老师的项目，clone 的是**自己那一份**，所以 `push` 只会推到孩子自己的仓库里，绝不会弄乱老师的原版。最后通过 **PR** 把成果交给老师审阅——这是最安全、也是最真实的协作方式。

> 目标是让孩子**亲手敲命令**、体会完整流程；大人只在验证身份这类步骤上搭把手就好。😊
