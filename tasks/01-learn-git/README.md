# 🎮 任务 01：学习 Git 和 GitHub

> 这是一个**闯关游戏**！一共 7 关，一关一关来，你就学会 Git 啦。

## 🎯 这次你会学会

- 用 **Git** 把自己的作品「存档」📀
- 用 **GitHub** 把作品「传到网上」☁️
- 学会程序员每天都在用的几个魔法命令 ✨

## 🤔 先搞懂两个词

**Git（读作「吉特」）** 是什么？

> 你玩游戏时会「**存档**」对吧？这样万一失败了，还能读回上一次的进度。
> Git 就是给你的文件做「存档」的工具。每存一次档，就叫一次「**提交（commit）**」。

**GitHub（读作「吉特哈brew」）** 是什么？

> Git 的存档一开始只在你自己的电脑上。
> GitHub 是一个网站，可以把你的存档**传到网上**，这样在别的电脑也能看到，还能和小伙伴一起分享、一起做项目。就像把照片传到网上的相册一样 📸。

## 🧰 开始前，先准备好

1. 电脑上已经装好了 **Git**。在终端（黑黑的命令窗口）里输入下面这行，能看到版本号就说明装好了：

   ```bash
   git --version
   ```

2. 有一个 **GitHub 账号**（网址：https://github.com ）。没有的话，请大人帮忙注册一个。

3. 打开「**终端**」（Terminal），用 `cd` 命令进到我们这个任务的文件夹：

   ```bash
   cd tasks/01-learn-git
   ```

> 💡 小提示：命令里的字母、空格、符号都要一模一样哦，就像抄口令一样。

## 🏁 关卡 1：做一张「自我介绍卡片」

先在 `my-work/` 文件夹里，新建一个文件叫 `about-me.md`，写上你的自我介绍。

可以照着下面这个样子写（换成你自己的内容）：

```markdown
# 你好，我是小明！👋

- 🎂 我今年 9 岁
- 🎨 我最喜欢：画画、踢足球
- 🐶 我最喜欢的动物是：小狗
- 🌟 我长大想当：宇航员
```

写好保存！这就是你这次要提交的**作品**。🎉

---

## 🏁 关卡 2：看看 Git 现在的状态

在终端里输入这个命令，问问 Git：「现在有什么新变化呀？」

```bash
git status
```

你应该会看到 Git 说：有一个新文件 `about-me.md` 还没有被保存（它可能显示成红色）。

> 🔍 `git status` 就像一面镜子，随时告诉你「哪些东西改了、哪些还没存档」。这是最常用的命令，随时都可以敲一敲看看情况！

---

## 🏁 关卡 3：把作品「存档」（add + commit）

存档要分两步：

**第一步：告诉 Git「我要存这个文件」**

```bash
git add about-me.md
```

再敲一次 `git status`，你会发现 `about-me.md` 变成绿色啦，说明它准备好被存档了。

**第二步：正式存档，并写一句「这次存了什么」**

```bash
git commit -m "我的第一张自我介绍卡片"
```

> ✏️ 引号里的话叫「**提交信息**」，用来记录你这次做了什么。就像给存档起个名字，以后一看就知道。

🎉 恭喜！你完成了人生第一次 **commit**！

---

## 🏁 关卡 4：开一条自己的「小路」（新建分支）

程序员做新东西时，喜欢先开一条自己的「**小路**」，在小路上做，不影响大家的「主路」。这条小路就叫「**分支（branch）**」。

新建并切换到一条叫 `my-card` 的小路：

```bash
git checkout -b my-card
```

用这个命令看看自己现在站在哪条路上：

```bash
git branch
```

前面有个 `*` 星号的，就是你现在所在的分支。应该是 `my-card`。🌿

> 🌳 想象一棵大树：`main` 是主干，`my-card` 是你新长出来的一根小树枝。你可以在小树枝上安心尝试，不怕弄坏主干。

---

## 🏁 关卡 5：把作品传到网上（push）

现在把你的存档「传到 GitHub 网上」。

> ⚠️ 这一步需要先在 GitHub 上有一个「仓库（repository）」，并和你的电脑连接好。如果还没弄，请大人帮忙，或参考本文件最下面的〈**大人帮忙区**〉。

连接好之后，输入：

```bash
git push -u origin my-card
```

传成功后，去 GitHub 网站上刷新一下，你就能看到自己的 `my-card` 小路和作品啦！☁️✨

> 🚀 `push` 就是「推上去」的意思——把你电脑里的存档，推到网上的仓库里。

---

## 🏁 关卡 6：建立一个 PR（Pull Request）

**PR** 的全名是 **Pull Request**，你可以理解成：

> 「我在自己的小路上做好了作品，想把它**合并到主路上**，请大家看一看、同意一下。」🙋

怎么建 PR：

1. 打开你的 GitHub 仓库网页。
2. 页面上一般会出现一个提示：**「Compare & pull request」**（比较并创建拉取请求），点它。
   - 如果没看到，就点上面的 **「Pull requests」** 标签 → 再点绿色的 **「New pull request」**。
3. 检查一下：是把 `my-card` 合并到 `main`。
4. 写个标题，比如：`加入我的自我介绍卡片`。
5. 点绿色按钮 **「Create pull request」**。

🎉 太厉害了！你建立了一个真正的 PR，这是程序员团队合作时**每天都在做的事**！

## 📔 魔法命令小抄（贴在墙上随时看）

| 命令 | 作用 | 一句话记忆 |
|------|------|-----------|
| `git status` | 看现在有什么变化 | 「照镜子」🪞 |
| `git add 文件名` | 挑出要存档的文件 | 「装进篮子」🧺 |
| `git commit -m "说明"` | 正式存档 | 「按下存档键」📀 |
| `git checkout -b 名字` | 新建并切换分支 | 「开一条小路」🛤️ |
| `git branch` | 看有哪些分支 | 「看看有几条路」🗺️ |
| `git push` | 把存档传到网上 | 「推上云端」☁️ |

## 👨‍👩‍👧 大人帮忙区（给家长/老师）

如果孩子是第一次用 Git，下面这些一次性设置可以由大人帮忙完成：

1. **设置 Git 的名字和邮箱**（只需设置一次）：
   ```bash
   git config --global user.name "孩子的名字"
   git config --global user.email "邮箱@example.com"
   ```

2. **在 GitHub 上新建一个仓库**（repository），拿到它的地址（比如 `https://github.com/用户名/pbl-learning.git`）。

3. **把本地项目和 GitHub 仓库连接起来**：
   ```bash
   git remote add origin https://github.com/用户名/pbl-learning.git
   ```

4. GitHub 现在需要用 **Personal Access Token** 或 **SSH key** 登录，第一次 `push` 时可能会要求验证身份，请协助孩子完成。

> 目标是让孩子**亲手敲命令**、体会完整流程；大人只在连接账号、验证身份这类一次性步骤上搭把手就好。😊
